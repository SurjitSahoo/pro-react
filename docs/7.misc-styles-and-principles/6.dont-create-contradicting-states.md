# Don't Create Contradicting States

```jsx showLineNumbers title="Here I forgot to update the error state on success and loading state in error case ðŸ’©ðŸ§¨ðŸ’£"
function Blog() {
    const [isLoading, setIsLoading] = useState(false);
    const [isError, setIsError] = useState(false);

    const fetchBlog = () {
        setIsLoading(true);
        fetch('api')
          .then(() => setIsLoading(false));
          .catch(() => setIsError(true));
    }
}
```

In the above code it's easy to create a situation where both `isLoading` and `isError` is true and vice versa.

function Blog() {

    const fetchBlog = () {
        status("loading");
        fetch('api')
          .then(() => setStatus("fulfilled"));
          .catch(() => setStatus("error"));
    }
}
